### FAQ-001: 微信小程序 WXML 文件编译错误

**发生时间：** 2026-05-19 22:39  
**问题类型：** 小程序编译错误  
**影响范围：** 微信小程序 `pages/backtest/backtest.wxml` 和 `pages/predict/predict.wxml`

---

#### 问题描述

在微信开发者工具中编译小程序时，出现 WXML 文件编译错误：

```
[ WXML 文件编译错误] ./pages/backtest/backtest.wxml
Bad value with message: unexpected token `.`.
  69 |       <view class="result-item highlight">
  70 |         <text class="result-label">总收益率</text>
> 71 |         <text class="result-value" style="color: {{result.total_return >= 0 ? '#ff4d4f' : '#52c41a'}}">
     |                                                                                                       ^
  72 |           {{result.total_return >= 0 ? '+' : ''}}{{result.total_return?.toFixed(2)}}%
  73 |         </text>
  74 |       </view>
at files://pages/backtest/backtest.wxml#71(env: Windows,mp,2.01.2510290; lib: 3.16.0)
```

---

#### 错误分析

微信小程序的 WXML 模板语言不支持以下 JavaScript 特性：

1. **可选链操作符 `?.`** - WXML 不支持 ES2020 的可选链语法
2. **可选链不能用于方法调用** - 如 `result.total_return?.toFixed(2)` 会导致编译错误

**受影响的文件：**

| 文件路径 | 问题代码 |
|----------|----------|
| `pages/backtest/backtest.wxml` | `result.total_return?.toFixed(2)` |
| `pages/predict/predict.wxml` | `result.predictions[0]?.trend` 等多处 |

---

#### 解决方案

使用 WXS（WeiXin Script）模块来处理数据格式化，WXS 是微信小程序专门设计的脚本语言。

**WXS 语法特点：**
- 可以在 WXML 中直接调用
- 不支持 ES6+ 语法（如可选链 `?.`）
- 需要使用传统的条件判断和空值检查

---

#### 修复步骤

##### 1. 修复 `pages/backtest/backtest.wxml`

**修复前：**
```wxml
<text class="result-value" style="color: {{result.total_return >= 0 ? '#ff4d4f' : '#52c41a'}}">
  {{result.total_return >= 0 ? '+' : ''}}{{result.total_return?.toFixed(2)}}%
</text>
```

**修复后：**
```wxml
<wxs module="filters">
/**
 * 格式化数值为固定小数位
 * @param {number} value - 要格式化的数值
 * @param {number} decimals - 小数位数，默认2位
 * @returns {string} 格式化后的字符串
 */
module.exports.toFixed = function(value, decimals) {
  if (value === null || value === undefined) return '0.00';
  decimals = decimals || 2;
  return Number(value).toFixed(decimals);
};

/**
 * 格式化收益率显示
 * @param {number} value - 收益率值
 * @returns {string} 带正负号的百分比字符串
 */
module.exports.formatPercent = function(value) {
  if (value === null || value === undefined) return '0.00%';
  var sign = value >= 0 ? '+' : '';
  return sign + Number(value).toFixed(2) + '%';
};
</wxs>

<text class="result-value" style="color: {{result.total_return >= 0 ? '#ff4d4f' : '#52c41a'}}">
  {{filters.formatPercent(result.total_return)}}
</text>
```

---

##### 2. 修复 `pages/predict/predict.wxml`

**修复前：**
```wxml
<text class="value" style="color: {{result.predictions[0]?.trend === 'up' ? '#ff4d4f' : '#52c41a'}}">
  {{result.predictions[0]?.price || '--'}}
</text>
```

**修复后：**
```wxml
<wxs module="filters">
/**
 * 获取数组元素
 */
module.exports.getItem = function(arr, index) {
  if (!arr || arr.length === 0) return null;
  return arr[index] || null;
};

/**
 * 获取对象属性值
 */
module.exports.getVal = function(obj, prop) {
  if (!obj) return '--';
  var val = obj[prop];
  return (val !== null && val !== undefined) ? val : '--';
};

/**
 * 获取趋势颜色
 */
module.exports.getTrendColor = function(trend) {
  return trend === 'up' ? '#ff4d4f' : '#52c41a';
};

/**
 * 获取趋势文字
 */
module.exports.getTrendText = function(trend) {
  return trend === 'up' ? '↑ 预测涨' : '↓ 预测跌';
};
</wxs>

<text class="value" style="color: {{filters.getTrendColor(filters.getItem(result.predictions, 0).trend)}}">
  {{filters.getVal(filters.getItem(result.predictions, 0), 'price')}}
</text>
```

---

#### WXS 使用说明

WXS（WeiXin Script）是微信小程序专属的脚本语言，与 JavaScript 类似但有限制：

| 特性 | WXS 支持 | JavaScript ES6+ |
|------|----------|-----------------|
| 可选链 `?.` | ❌ 不支持 | ✅ 支持 |
| 空值合并 `??` | ❌ 不支持 | ✅ 支持 |
| 箭头函数 | ❌ 不支持 | ✅ 支持 |
| 模板字符串 | ❌ 不支持 | ✅ 支持 |
| 传统函数 | ✅ 支持 | ✅ 支持 |
| 条件运算符 | ✅ 支持 | ✅ 支持 |

---

#### 修复后的效果

修复后，微信开发者工具中不再显示编译错误，页面可以正常显示：

- 回测结果页面显示收益率、年化收益等数据
- 预测结果页面显示预测价格和趋势

---

#### 相关文件

| 文件路径 | 修改内容 |
|----------|----------|
| `frontend/miniprogram/pages/backtest/backtest.wxml` | 添加 WXS 模块，使用 `filters.toFixed()` 和 `filters.formatPercent()` |
| `frontend/miniprogram/pages/predict/predict.wxml` | 添加 WXS 模块，使用 `filters.getItem()`, `filters.getVal()` 等 |

---

#### 总结

**问题原因：** WXML 模板不支持 ES6+ 的可选链操作符 `?.`

**解决方案：** 使用 WXS 模块封装数据格式化逻辑，通过 `filters.方法名()` 调用

**注意事项：** 
- WXS 中不能使用可选链 `?.`、空值合并 `??`、箭头函数等 ES6+ 语法
- 需要对 null 和 undefined 进行显式判断
- WXS 模块需要通过 `<wxs module="filters">` 标签引入

### FAQ-002: Web 前端找不到 /src/main.js 文件

**发生时间：** 2026-05-19 22:54
**问题类型：** 前端构建错误
**影响范围：** Web 前端项目 `frontend/web`

---

#### 问题描述

Vite 开发服务器启动后，浏览器控制台报错：

```
[vite] Pre-transform error: Failed to load url /src/main.js (resolved id: /src/main.js). Does the file exist?
```

**Vite 终端日志：**
```
VITE v5.4.21  ready in 326 ms
➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
22:54:00 [vite] Pre-transform error: Failed to load url /src/main.js (resolved id: /src/main.js). Does the file exist?
```

---

#### 错误分析

1. `frontend/web/index.html` 中引用了 `/src/main.js`
2. 但 `frontend/web/src/` 目录下不存在 `main.js` 文件
3. 之前在错误目录（`frontend/`）启动了 Vite，而不是正确的 `frontend/web/` 目录

---

#### 解决方案

1. 在正确的目录 `frontend/web/` 下启动 Vite
2. 创建缺失的 `src/main.js` 文件

**创建 main.js：**
```javascript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'
import router from './router'
import './style.css'

const app = createApp(App)
const pinia = createPinia()

app.use(pinia)
app.use(router)
app.use(ElementPlus)
app.mount('#app')
```

---

#### 相关文件

| 文件路径 | 操作 |
|----------|------|
| `frontend/web/src/main.js` | 新建 |
| 启动目录 | 从 `frontend/` 改为 `frontend/web/` |

### FAQ-003: Vue 路由未配置 - App.vue 使用 route.path 报错

**发生时间：** 2026-05-19 22:55
**问题类型：** Vue 路由错误
**影响范围：** Web 前端首页

---

#### 问题描述

```
Uncaught TypeError: Cannot read properties of undefined (reading 'path')
    at App.vue:12:36
```

---

#### 错误分析

1. `App.vue` 使用了 `<el-menu :default-active="route.path">` 和 `useRoute()`
2. 但项目中没有配置 Vue Router
3. 需要创建路由配置文件 `src/router/index.js`

---

#### 解决方案

**1. 创建路由配置文件 `src/router/index.js`：**
```javascript
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../pages/Home.vue'
import StockList from '../pages/StockList.vue'
import StockDetail from '../pages/StockDetail.vue'
import Backtest from '../pages/Backtest.vue'
import Predict from '../pages/Predict.vue'

const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/stock', name: 'StockList', component: StockList },
  { path: '/stock/:code', name: 'StockDetail', component: StockDetail },
  { path: '/backtest', name: 'Backtest', component: Backtest },
  { path: '/predict', name: 'Predict', component: Predict }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

**2. 更新 `main.js` 引入路由：**
```javascript
import router from './router'
app.use(router)
```

---

#### 相关文件

| 文件路径 | 操作 |
|----------|------|
| `frontend/web/src/router/index.js` | 新建 |
| `frontend/web/src/main.js` | 更新，添加路由 |

### FAQ-004: Vite 无法解析 @/api 模块导入

**发生时间：** 2026-05-19 22:56
**问题类型：** 模块解析错误
**影响范围：** 所有 Vue 页面组件

---

#### 问题描述

```
[vite] Pre-transform error: Failed to resolve import "@/api" from "src/pages/Home.vue".
```

页面组件中使用了 `@/api` 导入别名，但对应的 API 模块文件不存在。

---

#### 错误分析

1. Vue 页面中引用 `import { systemApi } from '@/api'`
2. `src/api/` 目录为空，缺少 API 模块文件
3. 需要创建完整的 API 模块结构

---

#### 解决方案

创建 API 模块文件：

**1. `src/api/index.js` - axios 实例配置：**
```javascript
import axios from 'axios'

const apiClient = axios.create({
  baseURL: '/api',
  timeout: 30000,
  headers: { 'Content-Type': 'application/json' }
})

apiClient.interceptors.request.use(
  config => config,
  error => Promise.reject(error)
)

apiClient.interceptors.response.use(
  response => response.data,
  error => { console.error('API Error:', error); return Promise.reject(error) }
)

export { systemApi } from './system'
export { stockApi } from './stock'
export { backtestApi } from './backtest'
export { predictApi } from './predict'

export default apiClient
```

**2. `src/api/system.js`：**
```javascript
import apiClient from './index.js'

export const systemApi = {
  health: () => apiClient.get('/system/health'),
  strategies: () => apiClient.get('/system/strategies'),
  info: () => apiClient.get('/system/info')
}

export default systemApi
```

**3. `src/api/stock.js`：**
```javascript
import apiClient from './index.js'

export const stockApi = {
  list: (params) => apiClient.get('/stock/list', { params }),
  detail: (code) => apiClient.get(`/stock/${code}`),
  kline: (code, params) => apiClient.get(`/stock/${code}/kline`, { params }),
  search: (params) => apiClient.get('/stock/search', { params })
}

export default stockApi
```

**4. `src/api/backtest.js`：**
```javascript
import apiClient from './index.js'

export const backtestApi = {
  run: (params) => apiClient.post('/backtest/run', params),
  history: (params) => apiClient.get('/backtest/history', { params }),
  detail: (id) => apiClient.get(`/backtest/${id}`)
}

export default backtestApi
```

**5. `src/api/predict.js`：**
```javascript
import apiClient from './index.js'

export const predictApi = {
  predict: (params) => apiClient.post('/predict', params),
  history: (params) => apiClient.get('/predict/history', { params })
}

export default predictApi
```

```javascript
const apiClient = axios.create({

apiClient.interceptors.response.use(
  response => response.data,
  error => { console.error('API Error:', error); return Promise.reject(error) }
)

// API 模块 - 不要在这里导入 apiClient！

export { predictApi } from './predict'
```

```
[Vue warn]: Failed to resolve component: Refresh
[Vue warn]: Failed to resolve component: TrendCharts
[Vue warn]: Failed to resolve component: DataAnalysis
[Vue warn]: Failed to resolve component: MagicStick
```

#### 错误分析

`App.vue` 中使用了 `<el-icon><Refresh /></el-icon>` 等 Element Plus 图标组件，但没有全局注册这些图标。

```javascript
import { createApp } from 'vue'

const app = createApp(App)
const pinia = createPinia()

// 注册所有图标
for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}

app.use(pinia)
app.use(router)
app.use(ElementPlus)
app.mount('#app')
```

### FAQ-008: backtestApi.getHistory 参数类型错误

**影响范围：** Web 前端首页

```javascript
const res = await backtestApi.getHistory({ limit: 5 })  // 正确：传入对象
```

### FAQ-009: systemApi.healthCheck 方法名不匹配

**发生时间：** 2026-05-19 23:05
**问题类型：** API 方法名不一致
**影响范围：** Web 前端首页系统状态显示

---

#### 问题描述

前端调用 `systemApi.healthCheck()`，但 API 模块中定义的方法名是 `health()`。

```
加载健康状态失败: TypeError: systemApi.healthCheck is not a function
```

导致系统状态一直显示 `error`。

---

#### 错误分析

`Home.vue` 调用：
```javascript
healthInfo.value = await systemApi.healthCheck()  // 方法名不一致
```

### FAQ-018: technical_factors.py DataFrame 布尔判断错误

**发生时间：** 2026-05-19 23:28
**问题类型：** Python 代码错误
**影响范围：** 预测功能 `/api/predict/predict`

---

#### 问题描述

```
ValueError: The truth value of a DataFrame is ambiguous. Use a.empty, a.bool(), a.item(), a.any() or a.all().
```

预测接口返回 500 错误。

---

#### 错误分析

`factors/technical_factors.py` 中使用了 `df = df or self.data.copy()` 来处理可选参数：

```python
def add_moving_averages(self, df: Optional[pd.DataFrame] = None) -> pd.DataFrame:
    df = df or self.data.copy()  # 这里会报错！
```

当 `df` 是一个 DataFrame 时，`df or self.data.copy()` 会尝试对 DataFrame 进行布尔判断，但 DataFrame 的布尔值是模糊的（一个 DataFrame 可能有多个行，无法直接判断 True 或 False）。

---

#### 解决方案

将所有 `df = df or self.data.copy()` 改为 `df = df if df is not None else self.data.copy()`：

```python
# 修复前
df = df or self.data.copy()

# 修复后
df = df if df is not None else self.data.copy()
```

共修复 9 处（包括 `add_moving_averages`, `add_macd`, `add_rsi`, `add_bollinger_bands`, `add_kdj`, `add_obv`, `add_atr`, `add_volatility`, `add_volume_indicators`）。

### FAQ-019: predict.py 数据预处理后数据为空导致 400

**发生时间：** 2026-05-19 23:32
**问题类型：** 数据处理逻辑错误
**影响范围：** 预测功能

---

#### 问题描述

预测接口返回 400 Bad Request，日志显示数据预处理失败。

---

#### 错误分析

`api/predict.py` 中的 `prepare_features` 函数使用了 `df_features.dropna()` 对整个 DataFrame 移除 NaN：

```python
# 修复前
df_features = factors.add_all_indicators()
df_features = df_features.dropna()  # 移除所有含 NaN 的行！

if len(df_features) < lookback:
    return None, None, None
```

由于 Baostock 返回的数据可能有一些非关键列包含 NaN，`dropna()` 后数据从 197 条变为 0 条，导致后续处理失败。

---

#### 解决方案

只对需要的特征列移除 NaN，而不是整个 DataFrame：

```python
# 修复后
feature_cols = ['sma_5', 'sma_20', 'sma_60', 'rsi_14', 'macd', 'macd_signal', 'macd_hist', 'kdj_k', 'kdj_d', 'kdj_j', 'bb_position_20', 'atr_14']
available_cols = [col for col in feature_cols if col in df_features.columns]

# 创建目标变量
df_features['future_return'] = df_features['close'].shift(-5) / df_features['close'] - 1
df_features['target'] = (df_features['future_return'] > 0.01).astype(int)

# 只对需要的列移除 NaN
df_features = df_features.dropna(subset=available_cols + ['target', 'future_return'])
```

修复后数据从 0 条变为 133 条，预测功能正常。
