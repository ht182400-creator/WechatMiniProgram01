#### useWebSocket Composable

**位置：** `frontend/web/src/composables/useWebSocket.js`

**核心能力：**
- **自动重连**：指数退避算法（3s → 4.5s → ... → 30s 上限）
- **订阅管理**：`subscribe(codes)` / `unsubscribe(codes)` 自动同步到服务端
- **心跳保持**：每 25 秒发送 ping，配合服务端 30 秒超时检测
- **响应式行情**：`quotes` ref 自动更新，驱动页面价格刷新
- **连接状态**：`connectionStatus` 三种状态：disconnected / connecting / connected
- **生命周期管理**：组件 `onUnmounted` 时自动清理连接和订阅

**使用示例：**
```javascript
import { useWebSocket } from '@/composables/useWebSocket'

const { connectionStatus, quotes, subscribe, unsubscribe } = useWebSocket()

// 订阅股票
subscribe(['600000', '000001'])

// 监听行情到达
watch(() => quotes.value, (newQuotes) => {
  // newQuotes['600000'] → { price: 7.85, change: 0.12, pct_change: 1.55, ... }
})
```

## 六、Bug 修复记录

### 6.1 Home.vue 变量提升错误修复（2026-05-21）

**问题：** 前端控制台报错：
```
Uncaught (in promise) ReferenceError: Cannot access 'watchlist' before initialization
    at Home.vue:356:29
```

**原因：** 在 `Home.vue` 中，3 个 `watch` 监听器（监 `connectionStatus`、自选股 codes 变化、`quotes` 行情刷新）内部引用了 `watchlist.value`、`hotStocks.value`、`selectedStock.value`，但这些 `watch` 写在了 `const watchlist = ref([])` 等状态变量声明**之前**。JavaScript 的 `const` 存在 **temporal dead zone**，声明之前无法访问。

**修复方式：** 将 3 个依赖状态变量的 `watch` 从文件顶部移到所有状态变量声明之后。修改后的声明顺序：

```
344: useWebSocket() 解构          ← 不依赖任何状态
346: 状态变量区                    ← watchlist, hotStocks, selectedStock 等
399: watch 响应式联动              ← 在状态变量之后，安全访问
476: 计算属性 / 格式化函数 / 生命周期
```

**修改文件：** `frontend/web/src/pages/Home.vue`

### 6.2 实时行情数据源优先级修复（2026-05-21）

**问题：** 后端日志显示 akShare 网络波动时，实时行情直接失败，没有回退到本地数据源：

```
[ERROR] [adapters.akshare_adapter] 获取实时行情失败: ('Connection aborted.', RemoteDisconnected('Remote end closed connection without response'))
```

同时其他接口（日线数据、指标）却正常从 `tdx_local` 获取数据，说明只有实时行情缺失了兜底逻辑。

**原因：** `DataSourceManager.get_realtime_quote()` 原先只尝试 `akshare` 一个数据源，失败就直接返回空 DataFrame，完全没有像 `get_daily_data()` 那样遍历多数据源兜底。而 `tdx_local` 适配器**已经实现了** `get_realtime_quote` 方法（从通达信本地 `.day` 文件读取最新日K数据作为近似实时行情）。

**修复方式：** 将 `get_realtime_quote()` 改为多数据源按优先级遍历模式：

```python
# 修改文件：backend/adapters/data_source_manager.py

def get_realtime_quote(self, codes: List[str]) -> pd.DataFrame:
    """获取实时行情（多数据源兜底，本地文件优先）"""
    if not codes:
        return pd.DataFrame()

    # 按优先级尝试各数据源（本地文件优先）
    for name in ['tdx_local', 'akshare', 'baostock', 'tushare']:
        if name in self.adapters:
            try:
                adapter = self.adapters[name]
                if not adapter.connected:
                    adapter.connect()
                df = adapter.get_realtime_quote(codes)
                if not df.empty:
                    logger.info(f"从 {name} 获取实时行情成功 ({len(df)} 条)")
                    return df
            except Exception as e:
                logger.warning(f"从 {name} 获取实时行情失败，尝试下一个数据源: {e}")

    logger.error(f"所有数据源均无法获取实时行情: {codes}")
    return pd.DataFrame()
```

**优先级顺序：** `tdx_local`（本地文件，零网络依赖）→ `akshare`（网络）→ `baostock` → `tushare`

**影响范围：**
- `/api/stock/realtime` API 端点
- `/ws/realtime` WebSocket 推送服务
- 两者均通过 `DataSourceManager.get_realtime_quote()` 获取数据，修复后自动生效

**修改文件：** `backend/adapters/data_source_manager.py`

### 6.3 通达信行情服务器适配器（mootdx 实时行情）（2026-05-21）

**问题：** `tdx_local` 适配器只能读取本地 `.day` 文件，盘中交易时段缺少当日数据文件，导致实时行情为空。akShare 等网络数据源稳定性差。需要一个能直连通达信行情服务器获取盘中实时数据的方案。

**解决方案：** 新建 `TdxHQAdapter`，通过 `mootdx.quotes.Quotes` 直连通达信行情服务器。

**架构设计：**

```
┌──────────────────────────────────────────────────────┐
│                   TdxHQAdapter                        │
│                    (tdx_hq)                           │
├──────────────────────────────────────────────────────┤
│  mootdx.quotes.Quotes.factory(                       │
│    market='std',                                     │
│    bestip=True,      ← 自动选最快服务器                │
│    multithread=True, ← 多线程吞吐量提升3倍             │
│    heartbeat=True,   ← 心跳保持长连接                  │
│    timeout=15        ← 弱网环境成功率提升              │
│  )                                                   │
├──────────────────────────────────────────────────────┤
│  实时快照 (quotes)     五档盘口+涨跌幅+PE/PB          │
│  K线数据  (bars)       1m/5m/15m/30m/时/日/周         │
│  分时数据 (minute)     当日分时价格线                  │
│  分笔成交 (transaction)逐笔成交明细                    │
└──────────────────────────────────────────────────────┘
```

**关键特性：**
- **延迟连接**：服务启动时不立即连接，首次请求行情时才建立连接，避免启动阻塞
- **字段归一化**：将 mootdx 原始字段（`vol`→`volume`, `reversed_bytes9`→`pct_change`）映射为标准字段
- **自动重连**：每次请求前检查连接状态，断连自动重连

**数据源优先级（实时行情）：**
```
tdx_hq（行情服务器，盘中实时五档） > tdx_local（本地文件，非交易时段兜底） 
  > akshare > baostock > tushare
```

**实测结果：**

### 6.4 第一阶段：tdx_local 升级为 mootdx.reader（2026-05-21）

**问题：** `tdx_local_adapter.py` 使用 `struct.unpack` 手动解析 .day 二进制文件（32字节记录），代码冗余（~200行），且无法自动处理指数/基金/债券的价格因子。preparereadme.MD 第2.1节明确推荐使用 `mootdx.reader`。

**解决方案：** 重写 `TdxLocalAdapter`，日K数据由 `mootdx.reader.Reader.daily()` 一站式读取。

class ParquetCache:
    def __init__(self, cache_dir):
        self._parquet_available = self._check_parquet_engine()  # 启动检测

    def set(self, key, data):
        if self._parquet_available:
            try:
                data.to_parquet(cache_path, index=True)  # 优先 parquet
                return True
            except Exception:

### 6.9 get_stock_list() DataFrame 列匹配修复（2026-05-21）

**问题：** AKShare 和 Baostock 适配器的 `get_stock_list()` 在某些环境下因返回 DataFrame 列数与预期不匹配而崩溃：

```
AKShare: "Length mismatch: Expected axis has 3 elements, new values have 2 elements"
Baostock: "Length mismatch: Expected axis has 5 elements, new values have 3 elements"
```

**原因：**
- **AKShare**：`stock_info_a_code_name()` 在不同版本中返回的列数可能变化（2-3列），`df.columns = ['code', 'name']` 硬编码赋值不兼容
- **Baostock**：`query_all_stock()` 在某些市场环境下返回的字段数可能少于预期的5个，`df.columns = ['code', 'name', 'ipo_date', 'out_date', 'stock_type']` 同样不兼容

**修复方式：**

**AKShare 适配器（`backend/adapters/akshare_adapter.py`）：**
```python
def get_stock_list(self) -> pd.DataFrame:
    df = self.ak.stock_info_a_code_name()
    if df.empty:
        return pd.DataFrame()
    # 防御性列名处理：取前两列作为 code 和 name
    if len(df.columns) >= 2:
        df = df.iloc[:, :2].copy()
        df.columns = ['code', 'name']
    elif len(df.columns) == 1:
        df.columns = ['code']
    df['source'] = 'akshare'
    return df
```

**Baostock 适配器（`backend/adapters/baostock_adapter.py`）：**
```python
def get_stock_list(self) -> pd.DataFrame:
    rs = bs.query_all_stock()
    data_list = []
    while rs.error_code == '0' and rs.next():
        data_list.append(rs.get_row_data())
    
    # 防御性处理：结果为空或无有效字段直接返回
    if not data_list or not rs.fields:
        return pd.DataFrame()
    
    df = pd.DataFrame(data_list, columns=rs.fields)
    expected_cols = ['code', 'name', 'ipo_date', 'out_date', 'stock_type']
    # 仅当实际列数与预期一致时才重命名
    if len(df.columns) == len(expected_cols):
        df.columns = expected_cols
    else:
        logger.warning(f"Baostock 列数不匹配: 期望 {len(expected_cols)} 实际 {len(df.columns)}")
        return pd.DataFrame()
    
    df = df[df['stock_type'].isin(['1', '2'])]  # 1=上交所, 2=深交所
    df['source'] = 'baostock'
    return df
```

**关键改进：**

| 适配器 | 修复前 | 修复后 |
|--------|--------|--------|
| AKShare | `df.columns = ['code', 'name']` 硬编码 | `iloc[:, :2]` 安全截取 + 列数检测 |
| Baostock | 无列数校验，直接赋值 | 列数匹配校验 + 空结果早期返回 |

**修改文件：**
- `backend/adapters/akshare_adapter.py`
- `backend/adapters/baostock_adapter.py`

### 6.14 StockList 行情页面预加载模式 + 中文名称（2026-05-21）

**问题：**
1. StockList 原来通过 API 调用 `stockApi.getList()` / `stockApi.search()` 加载数据，实时请求速度慢、容易挂起
2. 名称列显示股票代码而非中文名称

**修复方式：**

**1. 预加载静态数据：**
- 移除 `onMounted` 中的 `stockApi` 异步调用
- 内置 `ALL_STOCKS` 静态数组（约200只A股热门股票），包含 `code`、`name`（中文）、`market`、`industry` 字段
- 搜索和分页改为纯客户端 `computed` 计算：

```javascript
/** 搜索过滤（纯客户端计算） */
const filteredStocks = computed(() => {
  const kw = (searchKeyword.value || '').trim().toLowerCase()
  if (!kw) return ALL_STOCKS
  return ALL_STOCKS.filter(s =>
    s.code.includes(kw) ||
    s.name.toLowerCase().includes(kw) ||
    (s.industry || '').toLowerCase().includes(kw)
  )
})

/** 分页切片 */
const stocks = computed(() => {
  const start = (currentPage.value - 1) * pageSize
  return filteredStocks.value.slice(start, start + pageSize)
})
```

**2. 布局修复：**
- `max-width: 1400px` → `max-width: 100%`（解决页面过窄问题）
- 移除 `v-loading="loading"` 属性（无需加载状态）
- `.page-header` 从卡片样式改为简洁文字行（与其他页面统一）

**修改文件：** `frontend/web/src/pages/StockList.vue`
