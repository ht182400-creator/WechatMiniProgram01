# 量化数据中台 V4.0 - 完整架构设计文档

> 本文档是量化平台的完整架构蓝图，整合了 preparereadme.md 的需求愿景与当前项目实现状态。
> 版本：V4.0 | 更新日期：2026-05-21

---

## 1. 项目愿景

构建一个**本地部署、高可用、可扩展**的量化投研与行情展示系统，全面对标通达信/同花顺专业版：

### 核心目标

| 目标 | 说明 |
|------|------|
| **数据全覆盖** | A股全市场（股票、指数、ETF）的历史K线、实时行情、财务数据、股东股本、分红融资、研报评级、资金流向、热点题材 |
| **专业看盘界面** | K线图、五档盘口、分笔成交、财务指标表、杜邦分析、财务诊断 |
| **跨端支持** | Web端（Vue/ECharts）+ 微信小程序（原生+HQChart） |
| **生产级稳定性** | 多数据源自动容灾、任务调度、缓存策略、监控告警 |
| **良好扩展性** | 插件化数据源、配置化复权、支持新增因子与策略 |

## 3. 微服务详细设计

### 3.1 行情服务 (market-service:8001)

**核心接口：**

| 接口 | 方法 | 说明 | 数据来源 |
|------|------|------|----------|
| `/kline/{symbol}` | GET | K线数据（支持复权、多周期） | InfluxDB → mootdx回源 |
| `/realtime/{symbol}` | GET | 实时快照（五档盘口） | Redis缓存 |
| `/transaction/{symbol}` | GET | 分笔成交明细 | mootdx.transaction |
| `/ws/market` | WebSocket | 实时行情推送 | Redis订阅 |

**K线复权实现：**

### 4.2 PostgreSQL 表

```sql
-- 财务指标表
CREATE TABLE financial_indicators (
    id SERIAL PRIMARY KEY,
    code VARCHAR(10) NOT NULL,
    report_date DATE NOT NULL,
    report_type VARCHAR(10),  -- Q1/H1/Q3/Annual
    eps DECIMAL(10,4),        -- 每股收益
    net_asset DECIMAL(10,4),  -- 每股净资产
    roe DECIMAL(6,2),         -- 净资产收益率
    gross_margin DECIMAL(6,2),-- 毛利率
    net_margin DECIMAL(6,2), -- 净利率
    revenue DECIMAL(15,2),    -- 营业总收入
    profit DECIMAL(15,2),     -- 净利润
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(code, report_date)
);

-- 股东数据表
CREATE TABLE shareholders (
    id SERIAL PRIMARY KEY,
    code VARCHAR(10) NOT NULL,
    holder_name VARCHAR(200),
    holder_type VARCHAR(50),   -- 个人/基金/QFII/券商
    share_count DECIMAL(15,2),
    share_ratio DECIMAL(6,4),
    report_date DATE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 分红送转表
CREATE TABLE dividends (
    id SERIAL PRIMARY KEY,
    code VARCHAR(10) NOT NULL,
    announce_date DATE,
    plan_type VARCHAR(20),     -- 分红/送股/转增
    bonus_share DECIMAL(6,4),  -- 送股比例
    cash_dividend DECIMAL(6,4),-- 派息比例
    exec_date DATE,            -- 除权除息日
    created_at TIMESTAMP DEFAULT NOW()
);
```

```python
            'local': LocalFileAdapter(),     # 本地 .day 文件
            'mootdx': MootdxAdapter(),       # 通达信网络接口
            'akshare': AkshareAdapter(),     # akshare 数据源
            'tushare': TushareAdapter(),     # tushare pro
            'baostock': BaostockAdapter()    # baostock 备用
        }
        # 可配置的优先级顺序
        self.priority = config.get('priority', [
            'local', 'mootdx', 'akshare', 'tushare', 'baostock'
        ])

    def get_kline(self, symbol, start, end, frequency='daily'):
        """按优先级尝试各数据源"""
        for source_name in self.priority:
            try:
                data = self.sources[source_name].get_kline(
                    symbol, start, end, frequency
                )
                if data is not None and not data.empty:
                    logger.info(f"数据获取成功: {source_name}")
                    return data
            except Exception as e:
                logger.warning(f"Source {source_name} failed: {e}")
                continue
        raise DataSourceError("所有数据源均失败")
```

## 7. 微信小程序架构

### 7.1 BFF 层设计

```python
# bff-service/main.py
from fastapi import FastAPI, WebSocket
from redis.asyncio import Redis

app = FastAPI()
redis = Redis.from_url("redis://redis:6379")

@app.websocket("/ws/market")
async def market_websocket(ws: WebSocket):
    """WebSocket 实时行情推送"""
    await ws.accept()
    subscribed = set()

    while True:
        # 接收订阅请求
        data = await ws.receive_json()
        if data['action'] == 'subscribe':
            subscribed.add(data['symbol'])

        # 监听 Redis 行情变更
        async for msg in redis.pubsub():
            if msg['symbol'] in subscribed:
                await ws.send_json(msg)
```
