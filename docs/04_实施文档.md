### 1.2 容器化部署

```yaml
# docker-compose.yml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/stockquant
      - REDIS_URL=redis://cache:6379
    depends_on:
      - db
      - cache
    restart: unless-stopped

  frontend:
    build: ./frontend/web
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  db:
    image: postgres:15
    environment:
      POSTGRES_DB: stockquant
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  cache:
    image: redis:7-alpine
    restart: unless-stopped

volumes:
  postgres_data:
```

## 2. 环境配置

### 2.1 开发环境

```bash
# 1. 克隆项目
git clone <repository_url>
cd WechatMiniProgram01

# 2. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate   # Windows

# 3. 安装后端依赖
cd backend
pip install -r requirements.txt

# 4. 安装前端依赖
cd ../frontend/web
npm install

# 5. 初始化数据库
cd ../../backend
python scripts/init_db.py

# 6. 启动开发服务
# 终端1: 后端
uvicorn main:app --reload --port 8000

# 终端2: 前端
cd ../frontend/web
npm run dev
```

### 2.2 生产环境

```bash
# 1. 安装 Docker 和 Docker Compose
curl -fsSL https://get.docker.com | sh

# 2. 配置环境变量

# 编辑 .env 设置密码和密钥

# 3. 构建并启动
docker-compose -f docker-compose.prod.yml up -d --build
```

## 5. 监控与日志

### 5.1 日志配置

```python
# backend/config/logging.py
import logging
from logging.handlers import RotatingFileHandler

def setup_logging():
    log_format = '%(asctime)s [%(levelname)s] %(name)s: %(message)s'
    
    # 文件日志
    file_handler = RotatingFileHandler(
        'logs/app.log',
        maxBytes=10*1024*1024,  # 10MB
        backupCount=5
    )
    file_handler.setFormatter(logging.Formatter(log_format))
    
    # 控制台日志
    console_handler = logging.StreamHandler()
    console_handler.setFormatter(logging.Formatter(log_format))
    
    # 配置根日志器
    root = logging.getLogger()
    root.setLevel(logging.INFO)
    root.addHandler(file_handler)
    root.addHandler(console_handler)
```

## 6. 备份与恢复

### 6.1 数据库备份

```bash
# 备份 SQLite
cp data/stock_quant.db backups/stock_quant_$(date +%Y%m%d).db

# 备份 PostgreSQL
pg_dump -U user -d stockquant > backups/stock_quant_$(date +%Y%m%d).sql
```

### 6.2 自动备份脚本

```bash
# backups/backup.sh
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/path/to/backups

# 备份数据库
cp data/stock_quant.db $BACKUP_DIR/db_$DATE.db

# 备份缓存
tar -czf $BACKUP_DIR/cache_$DATE.tar.gz cache/

# 保留最近30天备份
find $BACKUP_DIR -mtime +30 -delete
```

## 7. 安全配置

### 7.1 Nginx HTTPS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    location / {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 7.2 防火墙配置

```bash
# ufw 配置
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp    # SSH
ufw allow 80/tcp     # HTTP
ufw allow 443/tcp    # HTTPS
ufw enable
```

## 9. 故障排除

### 9.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| API 响应慢 | 数据源API限流 | 启用缓存，增加重试 |
| 数据库连接失败 | 配置错误 | 检查 DATABASE_URL |
| 前端无法访问API | CORS配置 | 检查 main.py CORS配置 |
| 回测报错 | 数据不足 | 确保有足够的K线数据 |

### 9.2 日志分析

```bash

tail -f logs/app.log | grep ERROR

tail -f logs/app.log | grep "stock\|backtest"
```
